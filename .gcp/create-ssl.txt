manually create self-signed certificate on VM
https://www.digitalocean.com/community/tutorials/how-to-create-a-self-signed-ssl-certificate-for-apache-in-ubuntu-16-04

# generate cert
sudo openssl req -x509 -nodes -days 365 -newkey rsa:2048 -keyout /etc/ssl/private/apache-selfsigned.key -out /etc/ssl/certs/apache-selfsigned.crt

# create strong dh group
sudo openssl dhparam -out /etc/ssl/certs/dhparam.pem 2048
... and so on by example
After all instructions make sure proxy set up properly:
ProxyRequests On
ProxyPass / http://localhost:8081/
ProxyPassReverse / http://localhost:8081/

# Buy certificate
SSH to VM and execute: sudo openssl req -new -newkey rsa:2048 -nodes -out lifebook_pp_ua.csr -keyout lifebook_pp_ua.key -subj "/C=ua/ST=Cherkasy/L=Cherkasy/O=Lifebook/CN=lifebook.pp.ua/emailAddress=<email>"
visit ssl.com and paste generated .csr
copy .crt file to VM
make sure /etc/apache2/sites-available/default-ssl.conf contains correct paths to .csr and .key files
restart apache: sudo systemctl restart apache2
